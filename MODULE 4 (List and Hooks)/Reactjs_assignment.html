Q.1 Explain Life cycle in Class Component and functional component with Hooks
  • Class Components:
    • 1.Mounting Phase:
      • constructor(): Initializes state and binds event handlers.
      • render(): Returns JSX to be rendered.
      • componentDidMount(): Invoked immediately after a component is mounted. It's good for initializing data that needs a DOM presence.
    • 2.Updating Phase:
      • render(): Re-renders the component.
      • componentDidUpdate(prevProps, prevState): Invoked immediately after updating occurs. Good for interacting with the DOM after a component has been updated.
    • 3.Unmounting Phase:
      • componentWillUnmount(): Invoked immediately before a component is unmounted and destroyed. Good for cleanup such as removing event listeners or canceling network requests.
  • Functional Components with Hooks:
    • 1.Mounting Phase:
      • useState(): Initializes state.
      • useEffect(): Invoked after rendering. Can perform side effects like data fetching or DOM manipulation.
      • return JSX: Returns JSX to be rendered.
    • 2.Updating Phase:
      • useState(): Updates state.
      • return JSX: Re-renders the component.
      • useEffect(): Runs after every render. You can conditionally run effects based on state or props changes.
    • 3.Unmounting Phase:
      • useEffect() with cleanup: If you return a function from useEffect, it will run when the component is unmounted, similar to componentWillUnmount in class components.
  • Key Differences:
    • Class Components:
      • Use lifecycle methods like componentDidMount and componentDidUpdate.
      • More boilerplate code.
      • You need to bind event handlers manually.
      • Can hold local state.
    • Functional Components with Hooks:
      • useState(): Updates state.Use useEffect for side effects and state management with useState.
      • Less boilerplate code and cleaner syntax.
      • No need for manual binding of event handlers.
      • Can use React's Context API and custom hooks for complex state management.
